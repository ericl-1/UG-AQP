About

The Exam Results Analysis Tool automates MCQ item analysis and DIF reporting for UGME exams at the University of Ottawa Faculty of Medicine. Upload a QuestionMark or Scantron export and receive item statistics, score distributions, and formatted Word reports in seconds. For methodology details, see FAQ / Help or How This Works on the setup screen.

Getting Started
What files do I need before I can run an analysis? +
It depends on your data source:

QuestionMark (QM): You need two files — (1) the results file (.xlsx), exported directly from QuestionMark, and (2) the answer key file (.xlsx), prepared from the same export (see How do I prepare the answer key file? below). Do not modify either file before uploading.

Scantron: You need a single .txt file received from the scanning centre. No preparation is required — the answer key is embedded in the MASTER row of the file and will be read automatically. Switch the Data Source toggle in the Data Source & DIF section to Scantron before uploading.
Where do my results files come from? +
QuestionMark: Export your results report directly from QuestionMark as you normally would — no changes to the file are needed before uploading.

Scantron: Your results file is the .txt file received from the scanning centre after the exam is processed. No preparation is required — upload it as-is. The answer key is embedded in the file automatically.
How do I prepare the answer key file? +
Scantron: No preparation needed. The answer key is embedded in the MASTER row of the file received from the scanning centre and is read automatically on upload.

QuestionMark: The answer key comes from the same QuestionMark export as your results file. To prepare it:
  1. In the exported workbook, find the row for the correct answers entry — the one where all responses reflect the answer key rather than a real student's answers
  2. Copy that row to a new sheet within the same workbook (for your records), then copy or move that sheet to a separate new Excel workbook for upload
    • Ensure the column headers from the original file are included — the tool uses these to identify the answer columns
  3. Save and upload that file as your answer key in the Answer Key section
What if my exam has both English and French students? +
The tool handles bilingual exams automatically for both QuestionMark and Scantron. It uses stream codes to separate English and French students and compares their performance in the DIF analysis.

QuestionMark: By default the tool expects E for English and F for French in the results file stream column. If your export uses different codes, update them in the Language Codes field in the Data Source & DIF section before running the analysis.

Scantron: Students are identified by the course code they entered on their scan sheet. Enter the EN and FR course codes in the Data Source & DIF section so the tool can match each student to the correct stream. Because students enter these manually, some variation in codes is common — unmatched codes can be assigned to a stream at run time via the correction modal.
Can I use Scantron data instead of QuestionMark? +
Yes. In the Data Source & DIF section, switch the toggle from QM to Scantron. You then upload a single Scantron .txt export — no separate answer key file is needed, as the key is embedded in the MASTER row of the Scantron file. The rest of the analysis works identically to the QM path. When DIF is enabled, enter your EN and FR course codes in the Data Source & DIF section so the tool can assign each student to the correct language stream.
Some students have unrecognised course codes in my Scantron file — what happens? +
When unrecognised course codes are detected, the upload validation panel will show a warning listing each distinct unmatched code and how many students used it. There are two ways to fix this:

Quick fix (this session): When you click Analyze Exam, a correction modal will appear listing each affected student. Assign them to the correct stream using the dropdown, then click Save & Run Analysis. This applies to the current session only.

Global fix: If the mismatch is due to a typo in the EN or FR course code fields in the Data Source & DIF section, correct them there and click Re-validate in the preview panel to refresh the stream counts immediately — no re-upload needed.

Permanent fix: Correct the course codes in the source Scantron file and re-upload.
Pre-Analysis Options
Can I exclude a question before running the analysis? +
Yes — once your files are uploaded and the Analyze Exam button is active, a Question Exceptions link will appear just above the button. Click it to open the modal, which lists every question with its answer key letter. For each question you can set the disposition to Include (default), Credit, or Delete. A reason is required for any exclusion. You can also add alternate accepted answers — see below. Click Save Exceptions when done, then run the analysis normally.
What is the difference between Credit and Delete? +
Credit — students who answered correctly keep their point, and the question's point value is removed from the denominator for everyone. This effectively gives every student a small boost. The question's statistics are still shown in the analysis (so you can see how it performed), but it is excluded from flagging and DIF. The question still contributes to the exam average.

Delete — the question is removed entirely. No student receives a point for it, the denominator is reduced by 1 for everyone, and no statistics are calculated. It does not appear in flagging, DIF, or the exam average. It is shown as a greyed row labelled Deleted in the results table.
Can I accept more than one correct answer for a question? +
Yes — use the Alternate Key feature in the Question Exceptions modal. On any question row, click + Alt. Key to expand an inline panel showing answer choices A through E. The original key answer is pre-selected and greyed out for reference. Check any additional letters you want to accept, enter a reason, and click Save.

Students who answered the original key or any alternate answer will receive full credit. All statistics (p-value, discrimination, flagging) recalculate to reflect the updated scoring. The Key column in the results table will show the original answer followed by the alternates in parentheses — for example, A (+D). The reason you entered appears in the Comments column.
Will my exclusions be saved if I re-run the analysis? +
Yes — exclusions are saved in memory for the duration of your session. If you click Update Analysis after changing a setting, your saved exclusions will be applied again automatically. If you click Start Over or close the browser tab, exclusions are cleared along with everything else. They do not persist between sessions.
Can I change the colour theme of the app? +
Yes. Open Settings (hamburger menu, top right) and choose a theme from the Colour Theme dropdown under Appearance. Two options are available: uOttawa Theme (garnet, the default) and Elentra Theme (purple). Your choice is saved automatically and restored the next time you open the tool.
Data & Exports
What is the upload validation panel? +
After a successful upload, a parse preview panel appears below the upload fields. It confirms what the tool detected in your files before you run the analysis.

QuestionMark: The panel appears once both the results file and the answer key are uploaded successfully. It shows both filenames, student and question counts, EN/FR stream breakdown, the answer key for the first 10 questions, and a sample of the first 5 student rows. If the question count in the results file doesn't match the key, a mismatch warning appears. If any stream codes are unrecognised, the panel lists each distinct code and how many students used it.

Scantron: The panel appears after the single Scantron file is uploaded. It shows the same information — filename, counts, key preview, and student sample — with unmatched course codes highlighted if any are found.

In both cases this is a preview only — all students and questions are fully loaded regardless. You can fix the codes in the Data Source & DIF section and click Re-validate to refresh the stream counts without re-uploading, or click Re-upload if the file itself needs replacing. Click Looks good to dismiss the panel and proceed.
How do I save a record of the analysis for future reference? +
After running an analysis, open the hamburger menu (≡) on the analysis screen and select Save Session Record. This downloads a structured CSV file with three sections: an exam summary row (student counts, averages, reliability, flagged counts), one row per question with all statistics (p-value, discrimination, flag reason, disposition, EN/FR correct rates, DIF result), and the analysis thresholds in effect.

The file is named automatically using the exam title and date. It is designed to be imported into Excel, SharePoint, or other tools for longitudinal tracking. The Elentra ID column is included but blank for now — it will be populated in a future release once item ID linking is resolved.
Can I change the flagging thresholds? +
Yes — open Settings from the hamburger menu. The Analysis Thresholds section has sliders and input fields for all eight configurable thresholds, grouped into MCQ Item Flagging and DIF Analysis. Each group has its own Reset button, and a Reset all to defaults option is available at the bottom.

If you change any threshold after running an analysis, a warning banner will appear prompting you to re-run. The defaults match the UGME manual thresholds — only change them if instructed to do so.
Upload & Setup Issues
I uploaded the file but the tool says it can't read it — what do I do? +
The steps depend on your data source:

QuestionMark: Check that the file is saved as .xlsx (not .xls or .csv). The most common issues are a renamed sheet, a modified header row, or a file re-saved in a different format. Re-export directly from QuestionMark without modifying the file first.

Scantron: Check that the file is a .txt tab-delimited export and that it contains a MASTER row — this is where the tool reads the answer key from. If the MASTER row is missing or the file was exported in the wrong format, re-export from Scantron.

In both cases, click See details on the upload status indicator for a full diagnostic breakdown of what the tool found and what it expected.
The student count looks wrong after uploading — why? +
The tool automatically excludes rows that are not scoreable student responses — this includes header rows, timestamp rows, summary rows, and any students whose stream code doesn't match the EN or FR codes set in the Data Source & DIF section. If the count is unexpectedly low, check that your stream codes there match exactly what's in the file (case-sensitive). Click details after uploading for a full row-by-row breakdown.
My stream codes aren't E and F — where do I change them? +
In the Data Source & DIF section, update the EN and FR Language Codes fields to match the codes used in your QM export. The tool will re-read the uploaded file using the new codes the next time you run the analysis. Note: changing the codes after an analysis has been run will mark the results as stale and prompt you to re-run.
I uploaded the wrong file — can I replace it without starting over? +
Yes. Simply click the upload area again and select the correct file — it will replace the previous one. If an analysis has already been run, the tool will flag your results as potentially outdated and the Analyze Exam button will change to Update Analysis. Re-running the analysis will use the new file.
Running the Analysis
Do I need to enable DIF analysis, or can I skip it? +
DIF analysis is optional. Toggle it on or off in the Data Source & DIF section. When off, the DIF-related tabs, report sections, and KPI indicators are hidden. Turn it on if the exam has both English and French students and you need to identify questions that may have performed differently between streams. If your exam is a single-language cohort, you can leave DIF off.
What happens if I change a setting after running the analysis? +
The tool will detect the change and mark the current results as potentially outdated. An amber warning banner will appear — on the setup page and on the analysis page if you are already there — and the Analyze Exam button will change to Update Analysis. Your previous results are still accessible via View stale results, but they reflect the old settings. Re-run the analysis to generate fresh results.

This warning is triggered by: re-uploading a file, changing stream codes, toggling DIF on or off, or changing any analysis threshold in Settings. Changes to the exam title or date do not trigger it — they are cosmetic only.
I need to re-run the analysis with a corrected file — what's the process? +
Click ← Setup on the analysis screen to return to the setup page. Re-upload the corrected file in the Results Data or Answer Key section as needed. The Analyze Exam button will change to Update Analysis. Click it to re-run — all results, KPIs, and reports will refresh automatically.
What is the readiness panel above the Analyze Exam button? +
The Analysis Readiness panel appears as soon as you upload a file. It shows a compact summary of everything the tool knows before you run the analysis.

When all required files are loaded and no issues are detected, it displays a single compact green line: source, student count, question count, DIF status, and threshold profile.

When something needs attention — unmatched stream codes, custom thresholds active, or a missing answer key — it expands to a checklist. Problem items are highlighted in red with a ! indicator. Click the chevron to expand or collapse the detail view.

The feedback setup page has a similar panel that flags a default question ceiling (Q1–Q150) as a prompt to set it manually.
What does the status strip in the nav bar show? +
The session status strip appears in the results nav bar and shows at a glance what is loaded in the current session. It includes:

MCQ Analysis — Ready (analysis complete), Stale (settings changed since last run), or Not loaded.
DIF — On, Off, or Ready (DIF calculated).
Feedback — Ready or Not loaded.
Reports — Lists which report types are available to download (MCQ, DIF, Full, Feedback).

The strip updates automatically whenever an analysis is run or settings change. It is not shown on the Setup page.
Reading the Results
Some questions are flagged — do I need to do anything? +
Your role as a coordinator is to produce the report and pass it along — the exam director reviews flagged items and decides on action (retain, credit, delete, or change the key). The report you export already includes a recommended action for each flagged item. You don't need to make any decisions about individual questions yourself.
The reliability score is showing a warning — should I be concerned? +
A reliability warning (shown in the Reliability KPI) means the exam's internal consistency score (Cronbach's α) fell below the acceptable threshold of 0.70. This is noted in the report automatically. As a coordinator, you don't need to take action — simply include the report as-is. The exam director will interpret this result in context.
There are fewer students in the results than I expected — is something wrong? +
The tool only counts students whose stream code matches the EN or FR codes set in the Data Source & DIF section. Students with a different, blank, or unrecognised stream code are excluded. Check your language codes there, and click details under the upload confirmation for a diagnostic breakdown showing exactly which rows were included and which were excluded and why.
Exports & Reports
What reports are available and where do I find them? +
All reports are in the Reports tab. Use the report switcher at the top to select which report to preview, then click Download Word to export.

MCQ Report — item-level statistics, flagged questions, and too-easy questions. The standard report sent to exam directors.

DIF Report — differential item functioning analysis comparing English and French student performance. Only available when DIF is enabled.

Full Report — combines the MCQ and DIF reports into a single Word document. Only available when DIF is enabled.

Feedback Report — all student comments organised by question number in ascending order, with a summary and key findings section. Only available after running a feedback analysis. To preview any report before downloading, click the Preview link on its row — this opens a full-screen preview with an Export button inside. To download directly, click Export on the report row.
The Word document looks different on my computer — is that normal? +
Yes, this is expected. The reports are generated using the Calibri font and standard Word styles. If your computer doesn't have Calibri installed, Word will substitute a different font which may slightly affect spacing and layout. The content will always be correct. For best results, open the file in Microsoft Word (rather than Google Docs or LibreOffice).
Can I re-export a report without re-running the analysis? +
Yes. As long as the analysis results are still loaded (you haven't clicked Start Over or closed the browser tab), you can download the Word or CSV export as many times as you need from the relevant report tab. The analysis stays in memory for your entire session.
Can I save my results and come back to them later? +
Not currently — the tool runs entirely in your browser and does not save session data between visits. If you close the tab or browser, you’ll need to re-upload and re-run the analysis. Make sure to download your Word and CSV exports before closing. A session history feature is shown as a preview on the home screen and is planned for a future version.
Student Feedback
What is the Student Feedback feature? +
The Student Feedback tab lets you upload a QuestionMark feedback export (.xlsx) and automatically organise student comments by question number. The tool parses free-text comments, identifies which questions each comment refers to, and groups them into a structured report. You can review, reassign, and discard comments before exporting a Word report. If an MCQ analysis is also loaded in the same session, flagged questions are automatically cross-referenced in the feedback results.
What file do I need to upload for feedback? +
You need the QuestionMark feedback export (.xlsx) — the same raw export you receive from QuestionMark for the notes/feedback question. Upload it in the Feedback tab under the Student Feedback section. No preparation is needed before uploading.
Should I set a question count ceiling before running the feedback analysis? +
It is strongly recommended. Enter the total number of questions on the exam in the Question count field in the Feedback setup area before clicking Parse & Analyse Feedback. This tells the parser the valid range of question numbers (Q1 through your ceiling) and prevents it from picking up stray numbers in comments as question references.

If you have already run an MCQ analysis in the same session, the question count is set automatically and the field is locked. If you leave the field blank, the tool defaults to a ceiling of 150, which will produce a lower percentage in the Questions Reached indicator than reality.
What happens after I click Parse & Analyse Feedback? +
The tool parses every student comment and attributes each one to one or more question numbers based on the text. Comments that could not be attributed to a specific question are routed to a pending review screen where you can manually assign them to a question number, mark them as General, or discard them.

Once you have reviewed the pending items, click Apply & Generate Report. In standalone feedback mode, the tool takes you to the Feedback tab showing results by frequency. In combined analysis mode, validating feedback returns you to the setup screen and unlocks the Run combined analysis button — clicking it runs both MCQ and feedback together and lands you on the Item Analysis tab with all results populated.
Can I reassign or discard a comment after the report is generated? +
Yes — the By Question view shows every comment with controls to reassign it to a different question number, move it to General, or discard it. Discarded comments appear in the Discarded tab and can be recovered at any time. Changes are reflected immediately in the report preview and in the Word export.
What is the Feedback Report and how do I export it? +
The Feedback Report is a structured Word document listing all comments organised by question number, in ascending order. It includes a summary section, key findings (high and moderate volume questions), all comments per question, and any General feedback. To export it, go to the Reports tab and click Export on the Feedback report row. To preview it first, click the Preview link — this opens a full-screen view with an Export button inside.
What does the "Show comment categories in report" checkbox do? +
When checked (the default), comment category badges — Content, Translation, and Other — appear alongside each comment in the report preview and in the Word export. These badges are assigned automatically when the report is generated, and can be changed or cleared manually in the By Question view.

Unchecking the box hides all category badges from both the preview and the Word export without removing the underlying categorizations. This is useful when you want to share a clean report before the categorization review is complete. The checkbox appears inline on the Feedback report row in the Reports tab.
Troubleshooting
The app froze or showed an error after uploading — what should I do? +
First, try refreshing the page and re-uploading the file. If the error persists, click details under the upload area to see the diagnostic output — this will usually pinpoint the issue (unexpected column headers, missing sheet, incompatible file format, etc.). If you can't resolve it, take a screenshot of the error and the details panel and send it to elarouch@uottawa.ca along with the file you were trying to upload.
Something looks wrong in the report — who do I contact? +
Contact Eric Larouche at elarouch@uottawa.ca. Include the exam name, a description of what looks wrong, and if possible a screenshot or the exported file. The more detail you can provide, the faster the issue can be diagnosed.
